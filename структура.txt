Корневая папка (/web/portal/)
Файл/Папка	Назначение
index.php	Страница входа (логин/пароль). При успешной аутентификации создаёт сессию и перенаправляет на main.php.
main.php	Основной каркас сайта: левое меню и область контента. Определяет, какую страницу загрузить через параметр ?page=.... Проверяет права на просмотр каждого пункта меню.
auth.php	Подключается в начале каждого защищённого скрипта. Проверяет, авторизован ли пользователь, иначе редирект на index.php.
config.php	Центральный конфигурационный файл: подключение к БД, общие функции (hasPermission, cacheGet, getUserName), запуск сессии. Подключается везде.
logout.php	Завершает сессию и перенаправляет на страницу входа.
style.css	Единый файл стилей (тёмная пастельная тема).
script.js	Общие скрипты (например, подтверждение удаления).
.htaccess	Настройки Apache (запрет листинга папок, кэширование).
db_init.php	Скрипт для первоначальной инициализации БД (создание таблиц, базовых ролей и администратора). Удалить после первого запуска.
migrate_to_roles.php	Скрипт миграции для перехода на систему ролей (если обновляете старую версию).



pages/ – Страницы разделов
Каждый файл соответствует одному пункту меню. Название файла должно совпадать со значением параметра page в URL.

Файл	Раздел	Необходимые разрешения
dashboard.php	Главная / дашборд	view_dashboard
news.php	Новости	view_news
requests.php	Заявки	view_requests
equipment.php	Техника	view_equipment
users.php	Пользователи	view_users
links.php	Полезные ссылки	view_links
vacations.php	График отпусков	view_vacations
security.php	Информационная безопасность	view_security
birthdays.php	Дни рождения	view_birthdays
roles.php	Управление ролями	manage_roles


Важно: Каждый файл начинается с require_once __DIR__ . '/../config.php' и в начале проверяет права на просмотр с помощью hasPermission($pdo, 'view_...'). Если прав нет – выводится сообщение об отказе доступа.

handlers/ – Обработчики форм и действий
Все файлы в этой папке принимают POST/GET-запросы, выполняют действия (добавление, редактирование, удаление) и перенаправляют обратно на соответствующую страницу.

Именование: add_news.php, edit_news.php, delete_news.php, take_request.php, add_comment.php, upload_birthdays.php и т.д.

Каждый обработчик:

Подключает auth.php.

Проверяет конкретное разрешение (hasPermission($pdo, 'add_news')).

Выполняет валидацию данных.

Работает с БД.

Перенаправляет через header('Location: ...').

data/ – База данных
portal.db – файл SQLite. Права на запись должны быть у пользователя http.

cache/ – Кэш редко меняющихся данных
Создаётся автоматически. Используется в equipment.php и links.php для снижения нагрузки на БД.

uploads/ – Временные файлы
Сюда загружаются CSV-файлы с днями рождения, затем обрабатываются и удаляются.

templates/ – Шаблоны
birthday_template.csv – пример файла для загрузки дней рождения.


База данных (SQLite)
Основные таблицы:

Таблица	Назначение
users	Пользователи: id, username, password_hash, full_name, role_id
roles	Роли: id, name, description
permissions	Разрешения: id, name, description
role_permissions	Связь ролей и разрешений (many-to-many)
news	Новости
requests	Заявки
equipment	Техника
comments	Комментарии к технике
links	Полезные ссылки
vacations	Отпуска
security	Записи ИБ
birthdays	Дни рождения
Индексы созданы для ускорения запросов (например, idx_requests_status).


Ключевая логика в config.php
Функции, которые используются повсеместно:

hasPermission($pdo, $permission_name) – проверяет, есть ли у текущего пользователя указанное разрешение.

requireAuth() – вызывает редирект, если пользователь не авторизован.

cacheGet() / cacheSet() / cacheDelete() – простое файловое кэширование.

Важно: Сессия запускается только один раз благодаря проверке session_status().
